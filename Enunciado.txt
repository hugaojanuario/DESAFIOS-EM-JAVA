Desafio Completo: Sistema de Gestão de Recursos Humanos (HR)

1. Classes Base e Abstrata (POO Básico, Herança e @Override)

Crie as seguintes classes:

Classe A: Funcionario (Classe Base/Superclasse)

    Atributos (Encapsulamento):
        private String nome;
        private int matricula;
        protected double salarioBase; (Use protected para permitir acesso direto na subclasse)

    Construtor:
        Um construtor que inicializa nome, matricula e salarioBase.

    Getters & Setters:
        Crie Getters públicos para todos os atributos.
        Crie Setters públicos apenas para nome e salarioBase. A matricula não deve ser alterada.

    Método Essencial (Para ser Sobrescrito):
        Crie um método público chamado calcularSalarioTotal() que retorna um double.
        Na classe Funcionario, ele deve simplesmente retornar o valor de salarioBase.

    Método de Detalhes (Para ser Sobrescrito):
        Crie um método público chamado exibirDetalhes() (retorna void).
        Ele deve imprimir os detalhes básicos do funcionário (nome, matricula, salarioBase).





Classe B: Desenvolvedor (Subclasse)

    Herança: Esta classe deve herdar de Funcionario.

    Atributo Específico:

        private String linguagemPrincipal;

    Construtor:

        Crie um construtor que recebe nome, matricula, salarioBase (para o super construtor) e linguagemPrincipal.

    Sobrescrita (@Override):

        Sobrescreva o método calcularSalarioTotal(). Para Desenvolvedores, o salário total é o salarioBase mais um bônus fixo de R$ 500,00.

        Sobrescreva o método exibirDetalhes(). Deve chamar o método do pai (usando super.exibirDetalhes()) e depois imprimir a informação específica do Desenvolvedor (linguagemPrincipal).

Classe C: Gerente (Subclasse)

    Herança: Esta classe deve herdar de Funcionario.

    Atributo Específico:

        private int numEquipe; (Número de pessoas que gerencia)

    Construtor:

        Crie um construtor que recebe nome, matricula, salarioBase e numEquipe.

    Sobrescrita (@Override):

        Sobrescreva o método calcularSalarioTotal(). Para Gerentes, o salário total é o salarioBase mais um bônus de R$ 100,00 por membro da equipe (numEquipe).

        Sobrescreva o método exibirDetalhes(). Deve chamar o método do pai e depois imprimir a informação específica do Gerente (numEquipe).

2. Conceito static e Classe de Utilitários

Classe D: ContadorFuncionarios (Classe com static)

    Atributo static:

        private static int totalFuncionarios = 0;

    Método static:

        Crie um método public static void incrementarContador() que incrementa totalFuncionarios.

        Crie um método public static int getTotalFuncionarios() que retorna o valor de totalFuncionarios.

    Integração: Modifique os construtores das classes Desenvolvedor e Gerente para que, ao serem instanciados, chamem ContadorFuncionarios.incrementarContador().

3. Classe de Teste (Main) e Polimorfismo

Crie uma classe Main com o método main():

    Instanciação: Crie um objeto de cada tipo:

        Gerente g1 = new Gerente("Ana Souza", 101, 8000.00, 5);

        Desenvolvedor d1 = new Desenvolvedor("Beto Silva", 202, 5000.00, "Java");

    Teste static: Imprima o total de funcionários usando o método static: ContadorFuncionarios.getTotalFuncionarios().

    Teste @Override e Polimorfismo:

        Crie um array de Funcionario para armazenar todos os objetos (demonstração de polimorfismo):
        Java

        Funcionario[] quadroGeral = {g1, d1};

        Use um loop for-each para percorrer o array quadroGeral.

        Dentro do loop, chame ambos os métodos (exibirDetalhes() e calcularSalarioTotal()) para cada funcionário.

O objetivo final é ver que, ao chamar quadroGeral[i].calcularSalarioTotal(), o sistema executa a versão correta (sobrescrita) do método, seja ele para Gerente ou Desenvolvedor.

O que você vai aprender (Conceitos Chave):

    Encapsulamento (private): Protegendo dados internos.

    Modificadores de Acesso (public, private, protected): Controlando a visibilidade.

    Herança (extends): Reutilizando código (o funcionário genérico).

    super(): Chamando o construtor da classe pai.

    @Override: O compilador garante que você está, de fato, sobrescrevendo um método existente na classe pai. Ele é a garantia de que você está implementando a versão específica do método, de acordo com o tipo de objeto.

    Polimorfismo: Tratar objetos de classes diferentes (Gerente, Desenvolvedor) como um tipo comum (Funcionario) e ter o método correto executado em tempo de execução.

    static: Criando dados e métodos que pertencem à classe, não a uma instância específica. Útil para contadores e utilitários.
